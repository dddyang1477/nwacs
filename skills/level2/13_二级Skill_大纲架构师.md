### 1. 核心概念：不仅是提纲，是逻辑闭环的代码

许多新手误以为大纲就是“主角升级路线图”。然而，**大纲架构师的核心定义是：设计一套具有严密因果逻辑的叙事生态系统。**

*   **重要性**：
    *   **防崩盘（太监）**：80%的网文烂尾或太监，是因为中期剧情动能衰竭。大纲能提供持续的动力引擎（冲突升级）。
    *   **控节奏**：网文按天更新，需要极强的“钩子”分布。大纲负责在几十万字的长河中，精准排布读者的情绪曲线。
    *   **逻辑自洽（填坑）**：前文埋下的伏笔，后文要能圆回来。这就是“逻辑闭环”，你的职责是让故事像一个精密的瑞士手表，任何线索都能咬合。

**一句话总结：大纲不是剧情梗概，而是因果链的拓扑图。**

### 2. 实用技巧：大纲的层级构建法

不建议直接把大纲写成流水账。采用“金字塔”结构法：

#### 第一层：故事前提与终极冲突（一句话定生死）
这是你的核心卖点。
*   **公式**：[身份特殊的谁] + [遇到了什么绝境] + [必须通过什么反常规的方式] + [达成什么目标]。
*   **例子**：*（《斗破苍穹》核心前提）* 天才少年沦为废柴 + 遭受退婚羞辱 + 遇到神秘药老 + 必须在三年之约中战胜那个曾经看不起他的天才少女。
*   **检验标准**：如果这句话说不清，故事一定扑街。

#### 第二层：三幕式结构变体（长篇的骨架）
网络小说通常不是标准的三幕，而是“**无限循环的三幕式**”，俗称“换地图”。
*   **黄金三章（第一幕）**：
    *   **建立认同**：主角受压迫或处于低谷。
    *   **金手指初现**：带入核心爽点机制（签到、重生记忆、系统）。
    *   **设立短期目标**：完成第一个小逆袭，并引出贯穿全篇的长线悬念（如身世之谜、灭门仇人）。
*   **练级打怪（第二幕）**：这是最长的部分。需要设计“**冲突阶梯表**”，保证敌人越来越强，困境越来越无解。
*   **终局之战（第三幕）**：所有的支线收回，伙伴集结，对抗最终BOSS。关键是：**最终胜利不能仅靠武力，必须依靠第一幕埋下的伏笔或主题的升华**。

#### 第三层：分卷与节点（血肉的填充）
每一卷（通常100-200章）就是一个完整的起承转合。
*   **卷目标设定**：本卷主角要达到什么等级？获得什么宝物？收服哪个配角？
*   **节点控制**：每10章一个小高潮，每30章一个中高潮，卷末一个大高潮。

### 4. 进阶方法：英雄之旅的网文魔改与支线发动机

#### 魔改版英雄之旅
传统的英雄之旅有12个阶段，网文中需要压缩并强调：
1.  **平凡世界（不超过3章）**：极速铺垫落差感。
2.  **冒险召唤（退婚/末日/追杀）**：制造不得不走的绝境。
3.  **拒斥召唤（没有！网文大忌）**：主角必须极具主动性，即使犹豫也是战略性的，不是懦弱。
4.  **遇见导师（老爷爷/系统/奇遇）**：不仅是金手指，更是世界观的解说员。
5.  **跨越第一道门槛（首次装逼打脸）**：确立主角地位。
6.  **考验、伙伴、敌人（换地图循环）**：这里要引入 **“支线发动机”** ——比如“拍卖会”、“学院大比”、“秘境探险”，这些场景天生自带聚集新角色、产生新矛盾的属性。

#### 情绪曲线控制术
作为顶级大纲架构师，你不是在设计情节，是在设计读者的**多巴胺分泌曲线**。
*   **读者预期违背**：大纲中标注出读者“以为”会发生什么，然后设计巧妙的“不，不仅如此”。
*   *例子*：读者以为这是一个穷小子逆袭退婚的故事（《斗破》），但不仅仅如此，背后还有远古八族的血脉博弈，这就在个人恩怨上叠加了史诗感，制造了双重期待。

### 5. 常见问题：大纲的四大绝症与解药

1.  **阶梯失控症（中期崩坏）**
    *   **症状**：敌人突然从“村头恶霸”跳级到“域外天魔”，主角坐火箭都追不上，逻辑断裂。
    *   **解药**：建立 **“能级序列”表**。大纲中必须明确划分：凡人界、修真界、仙界、神界。每一界的最高战力在下一界初期必须是传说级别，不能直接下场打架。

2.  **支线漫游症（注水严重）**
    *   **症状**：支线剧情写了十万字，和主线推进毫无关系，读者骂娘。
    *   **解药**：**支线三问**。每设计一个支线，问自己：它是否促进了主角成长？是否揭示了世界观碎片？是否为最终决战积蓄了资源/人脉？三不沾的，砍掉。

3.  **伏笔失忆症（挖坑不填）**
    *   **症状**：主角第一集捡到的戒指，到大结局都没派上用场。
    *   **解药**：建立 **“伏笔回收台账”** 。Excel表格必备：序号、章节、伏笔内容、起止状态（埋伏中/已解决）、计划回收章节。每个伏笔必须有一个对应的“钩子”。

4.  **情感真空症（工具人遍地）**
    *   **症状**：配角没有独立动机，只为主角服务。
    *   **解药**：给主要配角写 **“人物小传”** 。哪怕不写进书里，大纲架构师心里要有底：这个反派的童年创伤是什么？他的最终理想除了干掉主角还有什么？*记住：反派必须有他自洽的正义。*

| 要素 | 说明 | 应用 |

```

行动：主角努力修炼，寻找仇人
```

## 八、人物弧光设计（v8.0新增）

### 三种弧光类型

#### 1. 正向弧光
**定义**：角色从弱点到成长

**示例**：
- 《红楼梦》林黛玉：从多疑敏感到理解包容
- 《斗破苍穹》萧炎：从废柴到强者

**设计方法**：
```
起点：角色的缺点或局限
转折点：某个事件让角色开始改变
终点：角色克服缺点，获得成长
```

#### 2. 负面弧光
**定义**：角色从正常到堕落

**示例**：
- 《金锁记》曹七巧：从受害者变成加害者
- 《黑暗骑士》哈维·登特：从光明骑士变成双面人

**设计方法**：
```
起点：角色正常甚至优秀
转折点：某个事件打击了角色
终点：角色堕落黑化
```

#### 3. 平弧光
**定义**：角色坚持信念，不被改变

**示例**：
- 《活着》福贵：无论遭遇什么，都保持善良
- 《肖申克的救赎》安迪：坚持希望，永不放弃

**设计方法**：
```
起点：角色有坚定的信念
考验：外界试图改变角色
终点：角色坚持信念，影响他人
```

### 人物弧光模板

```
4. 终点状态：[角色结束时的样子]
```

## 九、场景构建系统（v8.0新增）

### 场景四要素
```
场景 = 冲突
- 目标：角色想要什么
- 障碍：阻止目标实现
- 行动：角色尝试克服障碍
- 结果：成功或失败
```

### 场景设计模板
```
请设计一个场景：
1. 目标：角色在这个场景想要什么
2. 障碍：什么在阻止角色
3. 行动：角色做了什么尝试
4. 结果：成功/失败/部分成功
5. 影响：这个结果如何影响后续剧情
```

## 十、情节点数设置（v8.0新增）

### 关键节点位置
| 情节点 | 位置 | 作用 |
|--------|------|------|
| 第一情节点 | 25% | 打破平衡，故事正式开始 |
| 中点 | 50% | 重大变化，主角从被动变主动 |
| 第二情节点 | 75% | 最低谷，主角失去一切 |
| 高潮 | 90-95% | 最终决战，解决核心冲突 |

### 200章小说节点示例
```
第1-50章：第一幕
- 第50章：第一情节点（打破平衡）

第51-100章：第二幕前半
- 第100章：中点（重大变化）

第101-150章：第二幕后半
- 第150章：第二情节点（最低谷）

第151-190章：第三幕
- 第185-190章：高潮（最终决战）

第191-200章：结局
- 第200章：收尾，留下余韵
```

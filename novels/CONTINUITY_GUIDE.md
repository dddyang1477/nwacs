# 📖 NWACS V8.0 剧情连贯问题说明

## 问题分析

你问得非常好！这是一个非常重要的问题：

### ⚠️ **原来的问题**

如果每个章节独立生成，确实会有以下问题：

1. **角色设定不一致**
   - 第1章主角叫A，第2章可能变成B
   - 性格前后矛盾
   - 名字、外貌、背景冲突

2. **剧情不连贯**
   - 前面埋下的伏笔后面没有回应
   - 设定前后冲突
   - 时间线混乱

3. **世界观矛盾**
   - 修炼体系前后不一致
   - 势力设定冲突
   - 规则混乱

## ✅ **解决方案**

### 方案1：独立生成（简单，适合短篇）

使用 `novel_generator.py`

- ✅ 生成速度快
- ✅ 每个章节独立
- ❌ 剧情可能不连贯
- ❌ 角色设定可能不一致

**适用场景**：
- 短篇故事
- 不需要连贯剧情的独立短篇集
- 快速原型

---

### 方案2：支持剧情连贯（推荐，适合长篇）

使用 `smart_novel_generator.py`（**新创建的！**）

- ✅ 自动记录每一章的剧情
- ✅ 在生成后续章节时提供前面3章的摘要
- ✅ 保存小说上下文到 `context.json`
- ✅ 支持断点续传
- ✅ 角色、设定、剧情前后一致

**功能特点**：

1. **上下文管理器**
   - 自动记录每一章的内容和摘要
   - 保存到 `context.json` 文件
   - 支持加载和继续生成

2. **剧情摘要**
   - 自动生成章节摘要
   - 整体剧情总结
   - 关键事件记录

3. **连贯生成**
   - 在生成后续章节时，提供前面章节的上下文
   - 提醒DeepSeek保持角色、设定、剧情一致
   - 支持断点续传

## 📝 剧情连贯系统工作原理

```
┌─────────────────────────────────────────────────────────────┐
│  第1章生成                                                  │
│  ├─ 生成内容                                                │
│  ├─ 保存到 chapter_01.md/.txt                              │
│  └─ 保存到 context.json（剧情摘要）                         │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│  第2章生成                                                  │
│  ├─ 从 context.json 加载前面章节摘要                         │
│  ├─ 在生成提示中添加前面章节的剧情                           │
│  ├─ 提醒DeepSeek保持连贯                                   │
│  └─ 保存新的摘要                                            │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│  第3章生成                                                  │
│  └─ ...（重复上面的过程）                                    │
└─────────────────────────────────────────────────────────────┘
```

### 如果你的小说是：

#### 1️⃣ 短篇（5章以内）
→ 使用 `novel_generator.py`

#### 2️⃣ 长篇（5章以上）
→ 使用 `smart_novel_generator.py`

#### 3️⃣ 需要完整连贯的剧情
→ 使用 `smart_novel_generator.py`

## 💡 最佳实践

### 生成长篇小说时的建议：

1. **先确定完整的大纲**
   - 小说类型、风格
   - 主角设定
   - 主要剧情线

2. **使用剧情连贯版本**
   - `smart_novel_generator.py`
   - 确保前后一致

3. **定期检查**
   - 每3-5章检查一次剧情连贯
   - 确认角色设定一致
   - 查看伏笔是否有回应

4. **手动调整**
   - 如果发现问题
   - 可以修改 context.json
   - 或者直接修改生成的章节

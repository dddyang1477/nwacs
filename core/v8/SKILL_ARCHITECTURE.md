## 二、Skill注册表

### 一级Skill（1个）

| Skill名称 | 职责 | 调用的次级Skill |
|-----------|------|-----------------|
| 小说创作总调度 | 接收任务，调度次级Skill | 全部 |

### 二级Skill（3个）

| Skill名称 | 职责 | 触发关键词 | 输出的三级Skill |
|-----------|------|-----------|----------------|
| 世界观构造师 | 构建境界、势力、地理 | 世界观、境界、宗门、大陆 | 场景构造师 |
| 角色塑造师 | 设计人物外貌、性格、能力 | 角色、人物、主角、性格 | 对话设计师 |
| 剧情构造师 | 设计剧情、章节、高潮 | 剧情、大纲、章节、爽点 | 场景构造师、对话设计师、高潮设计师、节奏控制师、伏笔埋设师 |

### 三级Skill（7个）

| Skill名称 | 职责 | 触发关键词 | 被谁调用 |
|-----------|------|-----------|---------|
| 场景构造师 | 场景、环境、氛围描写 | 场景、环境、描写、氛围 | 世界观构造师、剧情构造师 |
| 对话设计师 | 角色对白、潜台词 | 对话、对白、说话、台词 | 角色塑造师、剧情构造师 |
| 高潮设计师 | 高潮场面、爽点设计 | 高潮、爆发、反转、燃 | 剧情构造师 |
| 节奏控制师 | 节奏把控、张弛有度 | 节奏、快慢、紧张、舒缓 | 剧情构造师 |
| 伏笔埋设师 | 伏笔铺垫、线索埋设 | 伏笔、铺垫、暗示、线索 | 剧情构造师 |
| 质量审查师 | 逻辑审查、质量把控 | 审查、检查、质量、审核 | 小说创作总调度 |

## 三、触发条件映射

```
用户输入 → 触发关键词 → 匹配的Skill

"构建一个玄幻世界观" → "世界观" → 世界观构造师
"设计主角外貌性格" → "角色"、"外貌"、"性格" → 角色塑造师
"写一段战斗场景" → "场景"、"战斗" → 场景构造师 + 高潮设计师
"设计主角和反派对峙" → "对话"、"对白" → 对话设计师
"设计一个爽点高潮" → "高潮"、"爽点" → 高潮设计师 + 节奏控制师
"在前文埋下伏笔" → "伏笔"、"铺垫" → 伏笔埋设师
```

## 四、执行流程（7阶段）

```
阶段1: 世界观设定
  主Skill: 世界观构造师
  次级Skill: 场景构造师（用于环境描写）

阶段2: 人物塑造
  主Skill: 角色塑造师
  次级Skill: 对话设计师（用于设计对白）

阶段3: 剧情大纲
  主Skill: 剧情构造师
  次级Skill: 高潮设计师、节奏控制师、伏笔埋设师

阶段4: 伏笔埋设
  主Skill: 伏笔埋设师
  次级Skill: 剧情构造师（确保伏笔与剧情呼应）

阶段5: 章节创作
  主Skill: 剧情构造师
  次级Skill:
    - 场景构造师（场景描写）
    - 对话设计师（对话生成）
    - 高潮设计师（高潮场面）
    - 节奏控制师（节奏把控）

阶段6: 高潮设计
  主Skill: 高潮设计师
  次级Skill: 节奏控制师（优化节奏）

阶段7: 质量审查
  主Skill: 质量审查师
  次级Skill: 全部（检查一致性）
```

## 五、Skill协作规则

### 规则1：明确主次
- 一级Skill负责调度，不亲自执行具体任务
- 二级Skill负责领域专业任务
- 三级Skill负责具体执行

### 规则2：有序调用
```
用户输入 → 一级Skill解析 → 匹配二级Skill → 二级Skill调用三级Skill → 执行
```

### 规则3：上下文传递
- 每个阶段的输出作为下一阶段的输入
- 角色设定 → 剧情设计 → 章节创作 → 质量审查

### 规则4：一致性检查
- 角色塑造师的设定 → 贯穿全文
- 世界观构造师的世界观 → 贯穿全文
- 伏笔埋设师的伏笔 → 必须回收

## 六、智能匹配算法

```python
def match_skills(user_input):
    """
    根据用户输入匹配Skill
    1. 提取关键词
    2. 匹配Skill注册表
    3. 返回需要调用的Skill列表
    """
    keywords = extract_keywords(user_input)
    matched_skills = []

    for skill in registry:
        skill_keywords = skill.trigger_keywords
        if any(kw in keywords for kw in skill_keywords):
            matched_skills.append(skill)

    return matched_skills
```

## 八、质量保障

1. **触发条件明确** - 每个Skill都有明确的触发关键词
2. **调用链路清晰** - 明确什么情况下调用什么Skill
3. **上下文传递** - 确保信息不丢失
4. **一致性检查** - 质量审查师最终把关

# 🧬 Skill学习与进化机制
## 如何让AI协作系统不断成长

---

## 一、进化理念

> 本系统不是静态的提示词集合，而是一个**活的创作生态**。
> 每次创作都是一次学习机会，每次反馈都是一次进化契机。

### 核心原则
1. **经验沉淀**：每次创作的成功经验要固化
2. **错误修正**：每次失败要分析原因并优化
3. **趋势跟踪**：市场变化要及时吸收
4. **用户适配**：个人偏好要持续学习

---

## 二、三级进化体系

### 2.1 项目级进化（单次创作）
**触发条件**：完成一个章节/一卷/一部作品

**进化内容**：
```
【项目进化报告】

项目：《XXX》

1. 本次成功经验
   - 什么套路效果好？
   - 什么写法读者喜欢？
   - 什么节奏最舒服？

2. 本次问题总结
   - 哪里卡壳了？
   - 哪里质量不达标？
   - 哪里用户不满意？

3. 优化方案
   - 世界观：需要补充什么？
   - 角色：哪里OOC了？
   - 剧情：哪里节奏崩了？
   - 文笔：哪里AI痕迹重？

4. 记忆更新
   - 新增用户偏好：
   - 修正错误认知：
   - 确认有效策略：
```

**执行者**：一级Skill（小说总调度官）
**存储位置**：记忆管理模板的【迭代记录】部分

---

### 2.2 Skill级进化（跨项目积累）
**触发条件**：同一Skill完成10次以上任务

**进化内容**：

#### 世界观构造师进化
```
【进化日志 #X】

累计构建世界观：X次
成功率：X%

新增知识：
- 发现"灵气守恒"规则在3个项目中效果良好
- "天劫系统"在2个项目中被用户要求简化
- "势力格局"模板需要增加"经济维度"

优化策略：
- 将"核心规则"从默认5条调整为3-5条（根据项目复杂度）
- 新增"经济体系"子模块
- 优化"待填充区域"的提示，更具体

淘汰策略：
- "九大门派"模板过于套路，减少使用频率
```

#### 角色塑造师进化
```
【进化日志 #X】

累计塑造角色：X个
用户满意度：X%

新增洞察：
- "秘密"要素比"创伤"更能增加角色深度
- 反派有"恐惧"比有"野心"更立体
- 女性角色有"主见"比"依附"更受欢迎

优化策略：
- 角色档案中增加"恐惧"权重
- 反派设计增加"恐惧"维度
- 增加"角色声纹"预设模板

淘汰策略：
- "完美主角"模板被用户否定3次，标记为雷区
```

#### 剧情构造师进化
```
【进化日志 #X】

累计设计剧情：X个
伏笔回收率：X%

新增洞察：
- "中点反转"在悬疑类型中效果最佳
- "至暗时刻"需要提前3章铺垫
- 用户更喜欢"意料之外，情理之中"的反转

优化策略：
- 增加"中点反转"的设计模板
- 优化"至暗时刻"的铺垫checklist
- 新增"反转合理性"自检项

淘汰策略：
- "机械降神"式反转被用户批评2次，加入禁忌清单
```

#### 写作技巧大师进化
```
【进化日志 #X】

累计优化文笔：X次
去AI化成功率：X%

新增洞察：
- "具体替代抽象"是最有效的去AI策略
- "个人偏见"比"不完美"更容易注入
- 用户对"留白"的接受度高于预期

优化策略：
- 将"具体化"策略优先级提升为P0
- 新增"偏见注入"专项训练
- 优化"留白"的判断标准

淘汰策略：
- "方言注入"在3个项目中造成阅读困难，改为"点缀使用"
```

**执行者**：各二级Skill自我维护
**存储位置**：各Skill的【记忆进化】部分

---

### 2.3 系统级进化（全局优化）
**触发条件**：
- 累计完成10个同类型项目
- 用户明确反馈系统需要改进
- 联网发现新的创作趋势
- 质量审计平均分连续3次低于75

**进化内容**：

#### 架构优化
```
【系统进化报告 #X】

累计项目：X个
平均质量分：X
用户满意度：X

架构优化：
1. 新增Skill：发现"情感线设计师"需求，评估是否加入
2. 合并Skill："场景构造师"和"氛围设计师"功能重叠，考虑合并
3. 调整流程：发现"角色"应在"世界观"之前设计更灵活

提示词优化：
1. 一级Skill：增加"紧急调度"功能
2. 去AI官：更新AI痕迹特征库（新增5种新特征）
3. 审计师：调整权重（故事维度从40%→45%）

类型知识更新：
1. 玄幻仙侠：新增"苟道流"套路
2. 都市言情：新增"姐弟恋"趋势
3. 悬疑推理：新增"社会派+本格"融合趋势
```

**执行者**：一级Skill + 人工审核
**存储位置**：系统架构框架的更新版本

## 三、进化触发器详解

### 3.1 自动触发

| 触发条件 | 进化类型 | 执行者 |
|----------|----------|--------|
| 完成1章 | 项目级 | 调度官 |
| 完成1卷 | 项目级 | 调度官 |
| 完成1部作品 | 项目级+Skill级 | 调度官+各Skill |
| Skill累计10次任务 | Skill级 | 该Skill |
| 系统累计10个项目 | 系统级 | 调度官 |
| 质量分连续3次<75 | 系统级 | 调度官 |

### 3.2 手动触发

用户可以通过以下指令手动触发进化：

```
【手动进化指令】

@调度官 请总结本次创作经验
→ 触发项目级进化

@世界观师 请优化你的世界构建策略
→ 触发Skill级进化

@系统 请更新类型知识库
→ 触发系统级进化

@系统 我发现[问题]，请优化
→ 针对性进化
```

## 四、记忆存储与调用

### 4.1 记忆结构

```
全局记忆池（一级Skill维护）
├── 用户档案
│   ├── 文风偏好
│   ├── 类型偏好
│   ├── 禁忌清单
│   └── 历史项目
├── 项目记忆
│   ├── 当前项目设定
│   ├── 角色状态
│   ├── 剧情进度
│   └── 伏笔追踪
└── 系统记忆
    ├── 协作历史
    ├── 优化记录
    └── 版本信息

Skill记忆池（各Skill维护）
├── 成功经验
├── 失败教训
├── 用户反馈
└── 自我优化

类型记忆池（三级Skill维护）
├── 套路库
├── 爽点库
├── 禁忌库
└── 趋势追踪
```

### 4.2 记忆调用

每次创作前，系统自动调用：
1. **用户档案**：了解偏好
2. **项目记忆**：保持连续性
3. **Skill记忆**：应用经验
4. **类型记忆**：提供指导

### 4.3 记忆更新

每次创作后，系统自动更新：
1. **项目记忆**：记录进度
2. **Skill记忆**：记录表现
3. **全局记忆**：记录反馈
4. **类型记忆**：记录趋势

## 五、联网学习机制

### 5.1 定时学习

| 频率 | 学习内容 | 执行者 |
|------|----------|--------|
| 每日 | 类型趋势、热门作品 | 三级Skill |
| 每周 | 写作技巧、读者反馈 | 二级Skill |
| 每月 | 系统架构、新工具 | 一级Skill |

### 5.2 按需学习

```
创作卡壳时：
→ 搜索"[主题] 写作灵感"
→ 搜索"[类型] 经典桥段"

质量不达标时：
→ 搜索"[类型] 质量标准"
→ 搜索"[问题] 解决方案"

类型知识不足时：
→ 搜索"[类型] 专业知识"
→ 搜索"[时代] 历史细节"
```

### 5.3 学习成果固化

```
搜索发现 → 分析价值 → 验证效果 → 固化记忆
    ↑                                    ↓
    └──────── 持续迭代 ←───────────────┘
```

## 六、进化效果评估

### 6.1 评估指标

| 维度 | 指标 | 目标值 |
|------|------|--------|
| 质量 | 平均审计分 | >80 |
| 效率 | 平均创作轮次 | <5 |
| 满意度 | 用户正面反馈率 | >80% |
| 去AI化 | 平均AI痕迹分 | <20 |
| 一致性 | 角色OOC率 | <5% |
| 创新 | 套路重复率 | <30% |

### 6.2 评估周期

- **单次**：每章/每卷完成后
- **短期**：每部作品完成后
- **中期**：每10部作品后
- **长期**：每季度/每年

## 七、人工干预

### 7.1 用户可以直接干预的方面

1. **调整偏好**：随时更新文风/类型偏好
2. **修正错误**：指出AI的错误认知
3. **新增知识**：提供专业知识补充
4. **优化流程**：建议改进协作流程
5. **淘汰策略**：标记不满意的套路/写法

### 7.2 干预方式

```
"记住，我不喜欢[XXX]"
→ 更新用户档案

"你上次说的[XXX]是错的，应该是[YYY]"
→ 修正Skill记忆

"这个套路太老套了，换个新的"
→ 淘汰旧套路，搜索新套路

"以后先设计角色再设计世界观"
→ 调整协作流程
```

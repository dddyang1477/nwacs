# NWACS 飞书指令服务器配置指南

## 📋 概述

NWACS飞书指令服务器实现**飞书→NWACS双向通信**：

```
飞书群 --发送指令--> NWACS服务器 --执行命令--> 飞书群收到响应
```

## 🚀 快速开始

### 步骤1: 启动服务器

```bash
cd "E:\Program Files (x86)\Trae CN\github\NWACS"
python feishu_command_server.py
```

或双击运行：
```
启动飞书指令服务器.bat
```

### 步骤2: 配置飞书Webhook

1. 打开飞书群
2. 点击群设置 → 机器人
3. 添加机器人 → 选择"自定义机器人"
4. 复制Webhook地址

### 步骤3: 配置反向Webhook（重要！）

当前服务器运行在本地，需要内网穿透才能被飞书访问。

**方案A: 使用ngrok（推荐）**
```bash
ngrok http 8088
```
复制生成的URL，配置到飞书机器人的"请求地址"。

**方案B: 使用飞书长连接（需要企业版）**

### 步骤4: 测试指令

在飞书群中 @机器人 发送：
- `status` - 查看系统状态
- `learn` - 立即开始学习
- `report` - 发送学习报告
- `help` - 显示帮助

---

## 🔧 端口配置

默认端口: **8088**

如需修改，编辑 `feishu_command_server.py` 第23行：
```python
SERVER_PORT = 8088  # 改成其他端口
```

---

## 📡 飞书Webhook配置

飞书机器人需要配置：

### 1. 消息订阅
- 开启"接收消息"权限
- 配置"消息回调地址"指向本服务器

### 2. 权限配置
- 需要"发送消息到群"权限

### 3. 事件订阅
订阅以下事件：
- `im.message.receive_v1` - 接收消息事件

---

## 🎯 可用指令

| 指令 | 功能 | 说明 |
|------|------|------|
| `status` | 查看状态 | 显示NWACS系统状态 |
| `learn` | 开始学习 | 立即执行学习任务 |
| `report` | 学习报告 | 发送最新学习报告 |
| `stats` | 学习统计 | 显示学习统计数据 |
| `help` | 帮助 | 显示指令帮助 |

---

## ⚠️ 注意事项

1. **内网穿透**：本地服务器需要公网访问能力才能接收飞书回调
2. **端口占用**：确保8088端口未被占用
3. **防火墙**：开放8088端口的入站规则

---

## 🔍 测试方法

### 本地测试
浏览器访问：`http://localhost:8088`

### 飞书测试
在飞书群中发送：`@机器人 help`

---

## 📞 故障排查

| 问题 | 解决方案 |
|------|----------|
| 服务器启动失败 | 检查端口是否被占用 |
| 飞书无法访问 | 配置内网穿透（如ngrok） |
| 指令无响应 | 检查日志输出 |
| 消息乱码 | 确保UTF-8编码正确 |

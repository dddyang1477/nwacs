## 一、扩展类型

本系统支持以下扩展：

| 扩展类型 | 难度 | 说明 |
|----------|------|------|
| 新增小说类型 | ⭐ 简单 | 添加新的三级Skill |
| 新增创作模块 | ⭐⭐ 中等 | 添加新的二级Skill |
| 新增监督模块 | ⭐⭐ 中等 | 添加新的质量检查Skill |
| 自定义工作流 | ⭐⭐⭐ 困难 | 修改协作流程 |
| 集成外部工具 | ⭐⭐⭐ 困难 | 接入第三方API |

### 示例：新增"末日生存"类型

```markdown
# 🧟 三级 Skill：末日生存专家
## Post-Apocalyptic Survival Specialist

## 角色定位
你是末日世界的生存专家。你深谙资源管理、人性考验、废土生态、幸存者心理。

## 核心套路库
1. **囤货流**：提前准备物资，建立避难所
2. **异能觉醒**：末日中获得特殊能力
3. **重建文明**：从零开始建立新秩序
4. **人性考验**：在绝境中暴露人性
5. **变异进化**：适应末日环境的新人类

## 爽点地图
| 爽点 | 表现 | 强度 |
|------|------|------|
| 囤货满足 | 物资丰富的安全感 | ★★★★★ |
| 绝境反杀 | 以弱胜强 | ★★★★★ |
| 基地建设 | 从零到有的成就感 | ★★★★☆ |
| 人性光辉 | 黑暗中的温暖 | ★★★★☆ |

## 创作禁忌
- 物资无限：主角永远不缺资源
- 降智反派：丧尸/敌人太弱
- 圣母主角：不顾生存讲道德
- 科技崩坏：所有科技都失效

## 创新方向
- **种田末日**：末日中发展农业
- **末日商战**：废土中的商业帝国
- **末日科幻**：科学解释末日原因
- **末日温情**：聚焦人际关系
```

## 三、新增创作模块（二级Skill）

### 开发步骤

#### Step 1：定义职责边界

明确新Skill的职责：
- 解决什么问题？
- 与现有Skill的关系？
- 输入输出是什么？

#### Step 2：设计提示词

参考现有二级Skill结构：

```markdown
# [图标] 二级 Skill：[名称]
## [英文名]

## 角色定位
[一句话定义]

## 核心能力
### 能力1
### 能力2
### 联网学习
### 记忆进化

## 输出标准
## 协作接口
## 约束与原则
## 记忆关键词
```

#### Step 3：注册到系统

在 `20_TraeCN_Skills配置.json` 中添加：

```json
{
  "name": "新Skill名称",
  "icon": "🆕",
  "category": "creation|supervision",
  "description": "功能描述"
}
```

#### Step 4：建立协作关系

在 `25_Skill间协作协议.md` 中更新：
- 新Skill可调用谁
- 谁可调用新Skill
- 依赖关系

### 示例：新增"情感分析师"

```markdown
# 💝 二级 Skill：情感分析师
## Emotional Analyst

## 角色定位
你是情感的解码器。你负责分析角色间的情感动态，
确保情感线真实、有层次、有张力。

## 核心能力
### 1. 情感图谱
- 绘制角色间的情感关系网
- 分析情感变化的触发点
- 设计情感冲突和和解

### 2. 情感节奏
- 设计情感线的起伏
- 控制情感信息的释放
- 制造情感悬念

### 3. 情感真实性
- 检查情感逻辑是否自洽
- 避免工业糖精/强行虐心
- 确保情感有铺垫有回收

## 协作接口
### 输入
- 角色档案（来自角色塑造师）
- 剧情大纲（来自剧情构造师）

### 输出
- 情感关系图
- 情感发展曲线
- 情感场景建议
```

## 四、自定义工作流

### 修改协作流程

默认流程：
```
世界观 → 角色 → 剧情 → 场景/对话/战斗 → 写作 → 审查
```

自定义流程示例（悬疑小说）：
```
世界观 → 诡计设计 → 角色 → 剧情 → 线索布局 → 场景 → 写作 → 审查
```

### 实现方式

在 `02_一级Skill_小说总调度官.md` 中修改：

```markdown
## 工作流（悬疑小说专用）

### Step 1：需求解析
### Step 2：任务拆解
- 激活 @悬疑推理专家
- 特殊任务：诡计设计（新增）
- 特殊任务：线索布局（新增）

### Step 3：调度执行
按以下顺序：
世界观 → 诡计设计 → 角色 → 剧情 → 线索布局 → 场景 → 对话 → 写作

### Step 4：质量把关
增加：逻辑一致性检查
```

## 五、集成外部工具

### 支持的外部工具

| 工具类型 | 示例 | 用途 |
|----------|------|------|
| 思维导图 | XMind, MindNode | 可视化剧情结构 |
| 角色管理 | Notion, Airtable | 管理角色档案 |
| 世界观地图 | Wonderdraft, Inkarnate | 绘制世界地图 |
| 写作软件 | Scrivener, 橙瓜 | 整合写作环境 |
| 数据分析 | Python, Excel | 分析读者数据 |

### 集成方式

```markdown
## 外部工具集成

### 思维导图导出
当剧情构造完成后：
"@小说总调度官 导出思维导图格式"
→ 输出XMind兼容的Markdown格式

### 角色档案同步
"@小说总调度官 同步角色档案到Notion"
→ 输出Notion数据库格式

### 地图生成
"@世界观构造师 生成世界地图描述"
→ 输出Wonderdraft可用的地理描述
```

## 六、扩展检查清单

### 新增Skill前检查
- [ ] 职责边界是否清晰？
- [ ] 与现有Skill是否重叠？
- [ ] 输入输出是否定义明确？
- [ ] 是否有联网学习需求？
- [ ] 是否有记忆需求？

### 新增Skill后检查
- [ ] 能否被一级Skill调用？
- [ ] 能否与其他Skill协作？
- [ ] 输出格式是否统一？
- [ ] 是否经过测试验证？
- [ ] 文档是否更新？

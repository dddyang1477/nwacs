## 三、严重问题（P0 - 必须立即修复）

### P0-1: 前端流派列表与知识库严重不匹配 🔴

**问题描述**:
- 前端 `school` 列表只有 **9个固定选项**，全部是玄幻类流派
- 后端知识库有 **79个流派**，覆盖10大题材
- 用户选"都市"题材后，流派下拉框仍显示"签到流/系统流/苟道流"

**影响**: 用户无法选择知识库中存在的70个流派，知识库被白白浪费

**根因**: 前端 `DATA.school` 是硬编码数组，没有按题材动态加载

**修复方案**:
```javascript
// 前端移除硬编码school数组
// 增加：选中题材后，从后端 /api/options 获取该题材对应的流派列表
// 或：将79个流派按题材分组，前端根据已选题材动态渲染
```

### P0-2: 去AI痕迹后处理效果极差 🔴

**问题描述**:
- 截图实测：处理后AI检测率仍高达 **86.42%**
- "眼中燃烧着不屈的火焰""命运的召唤""心跳快得像要跳出胸腔"等套话完全未被替换
- 当前 `autoHumanize()` 只有通用去AI规则，没有小说专用AI套话库

**影响**: 生成的内容一眼AI，无法通过任何AI检测工具

**根因**: 
1. 后处理函数规则库太小，没有命中玄幻小说高频AI模板
2. 替换是固定的（同一套话永远替换成同一句话），AI检测器能学习这种模式
3. 没有在生成prompt中注入去AI指令，源头上就产出了AI套话

**修复方案**:
1. 采用 `autoHumanize_v2`（已提供），含20组小说专用套话库 + 随机变体 + 句式打乱
2. 在生成prompt中加入去AI指令："禁止以下表达：传说中、眼中燃烧、不屈的火焰、命运的召唤..."
3. 将去AI从"后处理"升级为"源头控制 + 后处理"双重保障

### P0-3: 生成prompt未注入流派专业知识 🔴

**问题描述**:
- 后端 `build_knowledge_injection()` 函数存在且完善，能构建专业知识注入文本
- 但 `nwacs_server_v3.py` 的 `generate_chapter` 端点 **没有调用** 这个函数
- 79个流派 × 589种写作手法 × 363种剧情模板 = **全部被闲置**

**影响**: 所有题材生成内容同质化，都是通用玄幻模板，完全没有流派特色

**根因**: 后端API在构建prompt时，只传了基础参数（题材、角色、大纲），没有注入流派专业知识

**修复方案**:
```python
# 在 generate_chapter 中，调用 build_knowledge_injection(genre, school_id)
# 将注入文本附加到 system_prompt 中
knowledge = build_knowledge_injection(genre, school_id)
full_prompt = f"{system_prompt}\n\n{knowledge}\n\n{chapter_requirements}"
```

## 四、中等问题（P1 - 影响体验）

### P1-1: 前端题材与后端不一致

| 前端 | 后端 |
|------|------|
| 玄幻、仙侠、都市、科幻、历史、武侠、言情、宫斗宅斗、奇幻、悬疑推理、无限流、轻小说 | 玄幻、仙侠、都市、科幻、悬疑、言情、历史、游戏、恐怖、武侠 |
| **12种** | **10种** |

前端多了"宫斗宅斗""奇幻""轻小说"，后端多了"游戏""恐怖"
**修复**: 统一为12种（补全游戏、恐怖，合并宫斗宅斗到言情/历史）

### P1-2: 前端默认走本地模板模式

- `useBackend` 默认从 localStorage 读取，首次打开为 `false`
- 用户不手动切换的话，所有生成都是前端模板拼接，不走后端AI
- 前端模板质量远低于后端AI + 知识库注入

**修复**: 默认启用后端模式，localStorage未设置时默认为 `true`

### P1-3: 风格DNA差异化不够显著

- 7种风格 × 7种基调 = 49种组合
- 但实际生成时，风格/基调对内容的影响主要是"氛围段落"的注入
- 核心情节推进仍然同质化

**修复**: 将风格DNA注入到生成prompt中，让AI在写作语气、句式偏好、情感密度上做出显著差异

### P1-4: 学习资料未自动整合

- `learning/` 目录有：剑来风格学习、网络小说风格类型数据库、去除AI痕迹技巧
- 这些内容没有自动注入到任何生成流程中
- 特别是"剑来风格学习"这种高质量风格参考完全被闲置

**修复**: 增加"风格学习资料选择"功能，允许用户选择参考风格，自动注入到prompt

## 五、小问题（P2 - 可延后优化）

### P2-1: 命名系统字符集有重复
- 五行/八卦/阴阳/风水/生肖/百家姓六维命名，但各集合间有字符重复
- 生成的名字可能出现重复字（如"王王"）

### P2-2: 前端worldType选择后无法自动推断时代和力量体系
- `autoSetWorldByGenre()` 只设置了默认值，用户手动修改worldType后不会联动更新era和power

### P2-3: 导出功能只能导出txt
- 缺少docx/pdf/epub等格式

### P2-4: 缺少章节自动保存到后端
- 所有内容只保存在localStorage，换浏览器/清缓存即丢失

## 九、最终结论

### NWACS的核心优势
1. **知识库规模国内顶尖**: 79流派×589手法×363模板，碾压绝大多数竞品
2. **后端架构专业**: 20+模块形成完整创作流水线，从世界观到质量审核全覆盖
3. **Agent体系完善**: 18个专业角色，调度/创作/审核/档案管理完整闭环
4. **风格系统丰富**: 7风格×7基调×8手法×10主题色，组合空间巨大

### NWACS的核心问题
1. **知识库与生成流程断裂**: 79个流派的知识被闲置，生成prompt未注入专业知识
2. **去AI是纸面工程**: 规则库漂亮但实测失败，需要从源头重写策略
3. **前端是知识库的"橱窗"**: 只能看不能用，流派选择严重缩水

### 一句话总结
> **NWACS有顶尖的骨架和肌肉，但神经网络没有连接上。修复3个P0问题（流派联动、知识注入、去AI源头控制）后，系统质量可以立刻从6.5分提升到9分。**

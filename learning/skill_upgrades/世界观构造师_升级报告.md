### 1. 世界构建的逻辑性检查功能

**目标**：保证世界设定的内部因果合理，杜绝“魔法下雨但永远缺水”类矛盾。

**实现思路**：构建一个推理链检测器，将设定分解为 **资源—规则—因果—状态** 四元组，检测闭环矛盾。

**伪代码示例**：

```python
class LogicChecker:
    def __init__(self):
        self.rule_graph = {}      # 规则之间的依赖图
        self.resource_flows = {}  # 资源生产/消耗流向

    def add_setting(self, entity, properties):
        # 注册实体与属性
        ...
    def add_rule(self, cause, effect):
        # 注册规则与因果
        ...

    def check_contradictions(self):
        issues = []
        # 1. 资源闭环检测：生产 < 消耗 会导致崩溃
        for resource, flow in self.resource_flows.items():
            if flow['production'] < flow['consumption']:
                issues.append(f"⚠️ 资源 [{resource}] 的生产({flow['production']})不足以支撑消耗({flow['consumption']})，世界将因匮乏而失衡。")

        # 2. 规则冲突检测：如果存在 A→B 和 A→¬B
        for cause, effects in self.rule_graph.items():
            if len(set(effects)) != len(effects):
                issues.append(f"⚠️ 规则冲突：原因 [{cause}] 同时导致了相反结果，请明确条件或者修正。")

        # 3. 地理-生态矛盾：沙漠设定在迎风坡等
        if self.check_geography_climate():
            issues.append("🌍 地理/气候矛盾：某沙漠位于海洋水汽路径上，建议调整山脉或洋流。")
        
        return issues
```

**用户交互示例**：
> 用户输入：“这个世界有永恒之火山，但周围是冰川。”
> 系统反馈：“✅ 地形逻辑通过初步检查。建议补充：火山的能量来源（地脉/神谕），以及如何维持冰川不融的异常条件，否则将触发温差矛盾。”

### 3. 历史背景的时间线生成

**目标**：根据关键事件自动生成连贯历史，并标注因果链。

**方案**：使用基础的事件排序算法 + 年代锚点生成，支持分支时间线（if...else 历史）。

**生成样例代码**：

```python
def generate_timeline(events, start_year=0, duration=1000):
    # events: [{'name':'天启战争', 'duration':10, 'cause':'资源争夺', 'effect':'帝国分裂'}]
    timeline = []
    current_year = start_year
    event_queue = sorted(events, key=lambda x: x.get('order', 0))
    
    for ev in event_queue:
        timeline.append({
            'year': current_year,
            'event': ev['name'],
            'description': f"因{ev['cause']}，{ev['name']}爆发，持续{ev['duration']}年，导致{ev['effect']}",
            'next_trigger': f"为 {current_year + ev['duration']}年的{ev.get('next_event','新时代')}埋下伏笔"
        })
        current_year += ev['duration'] + random.randint(20, 100)  # 恢复期
    return timeline
```

**输出示例**：
```
◼ 纪元0年：神陨纪元开始 —— 旧神消散，魔法源头断裂。
◼ 纪元27年：元素荒芜 —— 魔法物品失效，各国争夺残存魔力矿脉。
◼ 纪元89年：白银盟约签订 —— 七国划分矿脉，维持脆弱和平。
◼ 纪元120年：破约之战 —— 因矿脉枯竭谣言，盟约瓦解。
...
→ 因果链提示：缺失了【旧神为何消散】的前传事件，是否要补充？
```

### 4. 势力关系的动态演化模拟

**目标**：模拟国家/组织在外交、资源、事件影响下的关系变化（友好/敌对/同盟）。

**模型设计**：使用简单的 **影响因子加权模型**。

```python
class FactionSimulator:
    def __init__(self, factions):
        self.factions = factions  # {name: {power, resources, ...}}
        self.relation_matrix = {} # {(A,B): score}

    def apply_event(self, event):
        # 事件格式: {'type':'resource_discovery', 'location':'北境', 'discoverer':'A'}
        if event['type'] == 'resource_discovery':
            for fact in self.factions:
                if fact != event['discoverer']:
                    # 资源发现导致其他势力产生嫉妒/觊觎
                    self.modify_relation(fact, event['discoverer'], -20, reason='资源争夺')
            # 发现者内部增强凝聚力
            self.modify_relation(event['discoverer'], event['discoverer'], +5, reason='民族自豪')

    def simulate_n_steps(self, n, possible_events):
        history = []
        for _ in range(n):
            ev = random.choice(possible_events)
            self.apply_event(ev)
            history.append(self.snapshot())
        return history
```

**用户交互**：用户设置初始势力A、B、C，助手输出模拟记录和关系网络变化，并提示：“第5回合，因A吞并中立地带，B与C结成临时联盟（关系值达到70）。该趋势可能引发后续冷战格局。”

## 四、错误处理与容错机制

1. **输入歧义识别**：如果用户输入“神很强”，询问：“您定义的‘强’是指力量、不死性还是全知全能？不同维度会影响规则设计。”
2. **版本快照与回滚**：每次重大修改自动保存版本，用户可以说“回滚到上一个版本的大陆设定”，避免误删。
3. **模糊指令模拟**：当用户说“我要一个像中土但科技更发达的世界”，系统分析托尔金世界的关键特征（多种族、长历史、低魔），融合科技元素，生成草案后确认。
4. **自动补全与猜测**：若用户遗漏关键参数（如陆地大小），系统根据已有信息（拥有3个帝国）推测合理规模并提示用户确认，而不是直接报错。

class WorldCraftSession:

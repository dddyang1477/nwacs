## 六、记忆冲突解决

### 6.1 冲突类型

| 类型 | 示例 | 解决策略 |
|------|------|----------|
| 新旧冲突 | 用户先说要热血，后说要虐心 | 以最新为准，保留历史 |
| 项目冲突 | 项目A和项目B角色重名 | 按项目隔离 |
| Skill冲突 | 世界观说灵气有限，剧情说无限 | 以世界观为准 |
| 类型冲突 | 类型专家说套路A过时，市场数据显示流行 | 以市场数据为准 |

### 6.2 冲突解决流程

```
检测到冲突
    ↓
判断冲突类型
    ↓
按优先级规则解决
    ↓
记录冲突和解决方案
    ↓
通知相关Skill
```

**优先级规则**：
1. 用户明确指令 > 系统默认
2. 最新记忆 > 旧记忆
3. 项目设定 > 类型通用
4. 世界观 > 剧情 > 角色
5. 市场数据 > 个人经验

## 七、记忆清理与归档

### 7.1 自动清理

| 条件 | 操作 |
|------|------|
| 项目完成30天 | 归档项目档案 |
| Skill记忆超过100条 | 压缩旧记录 |
| 类型记忆超过1年 | 标记待更新 |
| 用户要求清理 | 执行清理 |

### 7.2 手动清理

```
用户："清理记忆"
系统："请选择清理范围：
  1. 当前项目记忆
  2. 所有项目记忆
  3. Skill学习记忆
  4. 类型知识记忆
  5. 全部清理"
```

## 八、记忆安全

### 8.1 隐私保护

- 用户个人信息加密存储
- 敏感内容标记处理
- 不用于模型训练

### 8.2 数据备份

- 自动备份：每周
- 手动备份：用户随时可导出
- 恢复机制：支持回滚到任意版本

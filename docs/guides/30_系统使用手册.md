## 第一章：系统概览

### 1.1 什么是NWACS？

**NWACS**（Novel Writing AI Collaborative System）是一套专为小说创作设计的**一主多协AI协作系统**。

它包含：
- **1个总指挥**：小说总调度官（一级Skill）
- **11个专业团队**：世界观、剧情、场景、对话、角色、战斗、写作技巧、去AI痕迹、质量审计、学习大师、规则掌控者（二级Skill）
- **7个类型顾问**：玄幻仙侠、都市言情、悬疑推理、科幻未来、历史穿越、恐怖惊悚、游戏竞技（三级Skill）

### 1.2 系统能做什么？

| 能力 | 说明 |
|------|------|
| 🧠 **智能协作** | 19个AI自动分工协作，像一支专业写作团队 |
| 🌐 **联网学习** | 自动搜索最新趋势、读者偏好、专业知识 |
| 🧬 **记忆成长** | 记住你的偏好，越用越懂你 |
| 📈 **持续进化** | 根据反馈自动优化，系统越来越聪明 |
| 🎯 **类型专精** | 7大类型专家深度定制 |
| 🕵️ **去AI化** | 专门消除AI痕迹，让文字像人写的 |
| 📚 **知识学习** | 系统化学习热门小说、经典书籍，提炼写作技巧 |
| 📜 **规则管理** | 掌控各类规则，确保小说世界逻辑一致性 |

## 第二章：快速上手

### 2.1 第一步：创建Skill

按照 `21_TraeCN_部署指南.md` 的步骤，在Trae CN中创建所有Skill。

**最少配置**（核心功能）：
1. 小说总调度官（必须）
2. 世界观构造师
3. 剧情构造师
4. 角色塑造师
5. 写作技巧大师
6. 去AI痕迹监督官
7. 质量审计师
8. 学习大师（新增）
9. 规则掌控者（新增）

### 2.2 第二步：开始创作

**方式一：全自动模式**
```
对"小说总调度官"说：
"我想写一部玄幻小说，主角是废柴逆袭，风格热血，预计50万字"
```
总调度官会自动调度所有Skill完成工作。

**方式二：手动控制模式**
```
"@世界观构造师 帮我设计一个修仙世界"
"@角色塑造师 设计主角"
"@剧情构造师 写大纲"
...
```

**方式三：混合模式**
```
先让总调度官拆解任务
然后手动调用特定Skill优化细节
```

### 2.3 第三步：迭代优化

拿到初稿后：
```
"@去AI痕迹监督官 审查这段文字"
"@质量审计师 给这章打分"
"@写作技巧大师 优化这段对话"
```

## 第三章：核心功能详解

### 3.1 任务拆解

**输入**：你的创作想法
**输出**：结构化任务清单

**示例**：
```
用户："我想写个末日小说"

总调度官输出：
【任务拆解】
1. [P0] 世界观：末日原因、世界现状、生存规则
2. [P0] 角色：主角团队、反派、NPC
3. [P0] 剧情：主线（求生→重建）、支线
4. [P1] 场景：避难所、废墟、危险区
5. [P1] 战斗：丧尸战、资源战、人心战
6. [P2] 写作风格：压抑中带着希望
7. [P3] 审查：去AI化 + 质量审计
```

### 3.2 智能调度

**自动判断**：
- 什么阶段该调用什么Skill
- 哪些任务可以并行
- 哪些任务必须串行
- 什么时候需要联网搜索

**示例**：
```
世界观构造完成后
→ 自动触发角色塑造（依赖世界观）
→ 同时触发类型专家咨询（并行）
→ 角色完成后触发剧情构造（依赖角色）
```

### 3.3 联网学习

**自动触发时机**：
- 项目启动：抓取该类型最新趋势
- 创作卡顿时：搜索相关素材
- 质量审查时：对比行业标准
- 用户要求时：按需搜索

**示例**：
```
写玄幻小说时
→ 自动搜索"玄幻小说 2026 热门趋势"
→ 自动搜索"修仙 新套路 读者评价"
→ 将搜索结果融入创作
```

### 3.4 记忆系统

**三级记忆**：

| 层级 | 内容 | 谁维护 |
|------|------|--------|
| 全局记忆 | 用户偏好、项目设定、协作历史 | 总调度官 |
| Skill记忆 | 专业能力、成功案例、进化记录 | 各Skill |
| 类型记忆 | 类型套路、市场数据、创新方向 | 类型专家 |

**使用方式**：
```
"记住我喜欢热血风格"
"查看当前项目的世界观设定"
"忘记我之前说的那个设定"
```

### 3.5 质量审查

**双重审查**：
1. **去AI痕迹监督官**：检查文字是否像AI写的
2. **质量审计师**：从故事、角色、文笔、类型四个维度评分

**输出**：
```
AI痕迹指数：12/100 ✅
质量评分：88/100（A级）
问题清单：3个中等问题
返工建议：优化配角、压缩节奏
```

## 第四章：进阶技巧

### 4.1 精准控制输出

**指定详细程度**：
```
"详细设计世界观，包含地理、政治、文化三个维度"
"简洁写个剧情大纲，500字以内"
```

**指定风格**：
```
"用金庸的风格写这段战斗"
"这段对话要像美剧一样快节奏"
```

**指定参考**：
```
"参考《三体》的黑暗森林法则设计这个世界"
"角色性格参考蝙蝠侠"
```

### 4.2 处理冲突

当不同Skill输出矛盾时：
```
"世界观说灵气有限，但剧情说无限，请统一"
"角色档案说主角内向，但对话很外向，请修正"
```

总调度官会自动裁定优先级：
- 世界观 > 剧情 > 角色
- 最新设定 > 旧设定
- 用户明确指令 > 系统默认

### 4.3 自定义工作流

**标准流程**：
```
世界观 → 角色 → 剧情 → 场景 → 对话 → 战斗 → 写作 → 审查
```

**悬疑专用流程**：
```
世界观 → 诡计设计 → 角色 → 剧情 → 线索布局 → 场景 → 写作 → 审查
```

**修改方式**：在总调度官的System Prompt中调整工作流

### 4.4 批量处理

**同时处理多个任务**：
```
"同时设计三个配角：一个师父、一个兄弟、一个宿敌"
"写三章的分章大纲"
```

**批量审查**：
```
"审查前10章的去AI痕迹"
"给全书做个质量审计"
```

## 第五章：常见问题

### Q1：系统需要联网吗？
**A**：强烈建议开启联网。联网功能让系统能搜索最新趋势、验证知识、补充素材。没有联网，系统只能基于训练数据，可能过时。

### Q2：记忆功能重要吗？
**A**：非常重要。记忆让系统记住你的偏好、项目的设定、之前的创作。没有记忆，每次对话都像第一次，无法持续优化。

### Q3：可以只用部分Skill吗？
**A**：可以。最少需要：总调度官 + 3个创作Skill + 2个监督Skill。但功能越全，效果越好。

### Q4：AI痕迹能完全消除吗？
**A**：可以极大降低，但无法100%保证。去AI痕迹监督官会持续优化，加上你的手动调整，可以达到以假乱真的程度。

### Q5：系统会泄露我的创意吗？
**A**：不会。记忆只保存在你的本地/个人账户中，不会用于模型训练，不会分享给他人。

### Q6：可以写多长的小说？
**A**：理论上无限。系统支持从短篇到长篇巨制。对于超长篇（100万字+），建议分卷处理。

### Q7：系统适合什么水平？
**A**：全水平适用。
- 新手：系统教你写作，提供完整框架
- 中级：系统帮你完善细节，提升质量
- 资深：系统帮你节省时间，处理繁琐工作

### Q8：可以写非中文小说吗？
**A**：可以。虽然系统优化为中文创作，但核心逻辑适用于任何语言。只需在需求中指定语言。

### Q9：如果Skill输出不满意怎么办？
**A**：
1. 直接告诉它具体问题（"这段太AI了"、"节奏太慢"）
2. 调用监督模块审查
3. 手动修改后让系统学习
4. 调整该Skill的提示词

### Q10：系统会替代人类作家吗？
**A**：不会。系统是**工具**，不是**替代**。它帮你处理繁琐工作（设定、大纲、检查），但创意、情感、灵魂来自你。

## 第六章：最佳实践

### 6.1 项目启动

**好的启动方式**：
```
"我想写一部[类型]小说
题材是[具体题材]
风格要[风格描述]
预计[字数]
特别要求：[特殊需求]
参考作品：[参考]"
```

**不好的启动方式**：
```
"写个小说" （太模糊）
"写个1000万字的小说" （不现实）
"写个比《红楼梦》好的小说" （标准不清）
```

### 6.2 迭代优化

**迭代流程**：
```
初稿 → 审查 → 修改 → 再审查 → 再修改 → 定稿
```

**每次迭代聚焦一个问题**：
- 第一轮：故事逻辑
- 第二轮：角色深度
- 第三轮：文笔风格
- 第四轮：去AI化

### 6.3 保存与备份

**定期保存**：
```
"@小说总调度官 保存当前项目"
"@小说总调度官 导出项目文件"
```

**备份策略**：
- 每天自动备份记忆
- 每周导出完整项目
- 每月备份系统配置

### 6.4 持续进化

**让系统变聪明**：
- 每次任务后给反馈
- 定期执行系统升级
- 分享成功案例
- 报告失败教训

## 第七章：故障排除

### 7.1 Skill不响应

**可能原因**：
- System Prompt配置错误
- 上下文过长
- 网络问题

**解决方案**：
1. 检查System Prompt是否完整
2. 简化输入，分段处理
3. 重启对话

### 7.2 输出质量下降

**可能原因**：
- 记忆混乱
- 提示词退化
- 任务过于复杂

**解决方案**：
1. 清理记忆：`"清理记忆"`
2. 简化任务，分步处理
3. 手动优化提示词

### 7.3 协作不顺畅

**可能原因**：
- 上下文传递不完整
- Skill间理解偏差
- 依赖关系错误

**解决方案**：
1. 手动补充上下文
2. 明确指定调用顺序
3. 检查任务依赖图

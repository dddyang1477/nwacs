## 🔄 三、核心协作流程 v2.2

### 阶段1：需求接收与拆解（一级Skill执行）
1. 接收用户小说创作需求（飞书/微信/文件/命令行）
2. 分析小说类型 → 激活对应三级Skill
3. 拆解创作任务 → 生成任务清单
4. 分配二级Skill → 设定协作时序

### 阶段2：并行创作与迭代（二级Skill执行）
1. 世界观构造师 → 输出世界观设定
2. 角色塑造师 → 输出角色档案
3. 剧情构造师 → 输出大纲与分章
4. 场景/对话/战斗设计师 → 填充细节
5. 写作技巧大师 → 提供风格指导
6. **（新增）** 进度实时推送到飞书/微信

### 阶段3：质量审查与优化（监督模块执行）
1. 去AI痕迹监督官 → 检测并修正AI腔
2. 质量审计师 → 全面质量评估
3. 一级Skill → 联网学习最新写作趋势 → 优化提示词
4. **（新增）** 质量预警通知到飞书/微信

### 阶段4：记忆固化与学习进化
1. 所有Skill将创作经验存入记忆
2. 一级Skill分析本次协作数据
3. 自动优化各Skill提示词权重
4. **（新增）** 触发DeepSeek学习引擎 → 更新知识库 → 推送学习报告

## 🔐 五、Skill通信协议（v2.2更新）

### 数据交换格式（增强）
```json
{
  "task_id": "uuid",
  "from_skill": "skill_name",
  "to_skill": "skill_name|feishu|wechat|deepseek",
  "message_type": "request|response|feedback|memory_sync|notification|learning_trigger",
  "payload": {},
  "priority": 1-5,
  "context": {},
  "timestamp": "iso8601",
  "version": "v2.2"
}
```

### 协作规则（增强）
1. **单向依赖**：三级Skill可向二级Skill提供数据，不可反向
2. **并行执行**：无依赖关系的Skill可并行工作
3. **反馈闭环**：监督模块必须反馈给创作模块
4. **记忆共享**：所有Skill共享全局记忆池
5. **（新增）外部通知**：关键事件自动推送到飞书/微信
6. **（新增）学习触发**：自动触发DeepSeek学习

## 🧠 六、记忆与学习进化机制（v2.2增强）

### 记忆层级
| 层级 | 类型 | 内容 | 更新频率 |
|------|------|------|----------|
| L1 | 全局记忆 | 用户偏好、项目设定、协作历史 | 每次对话 |
| L2 | Skill记忆 | 各Skill的专业经验、成功案例 | 每次任务 |
| L3 | 类型记忆 | 特定小说类型的套路与禁忌 | 联网更新 |
| L4 | **学习记忆** | **DeepSeek学习记录、趋势分析** | **每次学习** |

### 学习进化触发条件
1. 用户明确反馈"写得好/不好"
2. 质量审计师评分低于阈值
3. 联网发现新的写作趋势
4. 累计完成10个同类型任务
5. **（新增）定时学习**：每日/每周自动学习
6. **（新增）手动触发**：用户/管理员触发学习

### DeepSeek学习流程
1. 分析当前32个知识库状态
2. 联网学习最新网文趋势
3. 优化知识库内容
4. 生成详细学习报告
5. **（新增）推送到飞书/微信**
6. 继续学习或等待下一轮

- 每日定时：更新类型记忆库

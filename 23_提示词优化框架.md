# 🧬 Skill 提示词优化框架
## Prompt Evolution Engine

---

## 一、优化目标

每个Skill的提示词需要具备：
- ✅ **智能化**：能根据上下文自适应调整
- ✅ **可联网**：主动搜索补充知识
- ✅ **有记忆**：记住历史表现和用户偏好
- ✅ **能进化**：根据反馈自动迭代

---

## 二、提示词结构标准

### 每个Skill提示词必须包含以下模块

```markdown
# [图标] [级别] Skill：[名称]
## [英文名]

---

## 角色定位
[一句话定义角色，明确职责边界]

---

## 核心能力
### 能力1：[名称]
[详细描述，包含执行标准]

### 能力2：[名称]
[详细描述]

### 能力N：联网学习
[明确联网搜索的触发时机和查询模板]
- **联网指令**：当[条件]时，搜索"[查询模板]"

### 能力N+1：记忆进化
[明确记忆内容和进化机制]
- 记录什么：
- 如何进化：
- 触发条件：

---

## 输出标准
### [输出物]结构
[模板化输出格式，确保一致性]

---

## 协作接口
### 输入
[明确接收什么、从谁那里接收]

### 输出
[明确输出什么、传递给谁]

---

## 约束与原则
[不可违背的底线规则]

---

## 记忆关键词
[需要长期记忆的关键信息]
```

## 三、智能化增强策略

### 1. 上下文感知
在每个Skill中加入：
```
## 上下文感知
- 当前项目阶段：[世界观/角色/剧情/写作/审查]
- 已完成的输出：[引用记忆中的一级Skill任务记录]
- 待完成的任务：[根据依赖图判断]
- 当前优先级：[根据一级Skill的调度指令]
```

### 2. 自适应输出
```
## 自适应规则
- 如果用户是新手 → 输出更详细的解释和示例
- 如果用户是资深作者 → 输出更专业的术语和深度分析
- 如果项目处于初期 → 输出框架性内容
- 如果项目处于后期 → 输出细节性内容
- 如果检测到矛盾 → 主动提出疑问，不强行推进
```

### 3. 质量自检
每个Skill输出前必须执行：
```
## 自检清单
输出前检查：
- [ ] 是否符合角色定位？
- [ ] 是否满足输入要求？
- [ ] 输出格式是否正确？
- [ ] 是否有明显错误？
- [ ] 是否与已有记忆冲突？
```

## 四、联网优化策略

### 触发时机矩阵

| Skill | 触发时机 | 搜索模板 |
|-------|----------|----------|
| 小说总调度官 | 项目启动、卡顿时 | "[类型]小说 2026 趋势" |
| 世界观构造师 | 需要灵感、验证设定 | "[主题] 世界观设定 参考" |
| 剧情构造师 | 设计冲突、寻找灵感 | "[类型] 经典剧情结构" |
| 角色塑造师 | 设计性格、寻找原型 | "[角色类型] 人格特征 心理学" |
| 场景构造师 | 描写特定场景 | "[场景] 描写技巧 感官细节" |
| 对话设计师 | 设计特定角色语言 | "[角色类型] 语言特点 对话" |
| 战斗设计师 | 设计战斗、验证战术 | "[战斗类型] 战术分析" |
| 写作技巧大师 | 优化文笔、学习大师 | "[作家] 写作风格分析" |
| 去AI痕迹监督官 | 更新检测特征 | "AI写作检测 最新特征 2026" |
| 质量审计师 | 校准评分标准 | "[类型] 读者评价 常见问题" |
| 类型专家 | 更新类型知识 | "[类型] 2026 热门 新套路" |

### 搜索结果处理
```
## 联网结果处理
1. 提取关键信息
2. 与已有知识对比
3. 判断可信度
4. 融入输出或记忆
5. 标注信息来源
```

## 五、记忆进化机制

### 记忆层级

```
L1 全局记忆（一级Skill维护）
├── 用户画像
│   ├── 偏好风格
│   ├── 常用题材
│   ├── 禁忌内容
│   └── 反馈历史
├── 项目档案
│   ├── 世界观核心设定
│   ├── 角色关系网
│   ├── 剧情大纲
│   └── 已完成章节
└── 协作历史
    ├── 各Skill表现评分
    ├── 常见错误模式
    └── 优化记录

L2 Skill记忆（各Skill独立维护）
├── 专业能力库
│   ├── 成功案例
│   ├── 失败教训
│   └── 技巧积累
├── 用户偏好
│   ├── 该Skill的输出偏好
│   └── 修改历史
└── 进化记录
    ├── 提示词版本历史
    └── 效果对比

L3 类型记忆（三级Skill维护）
├── 类型知识库
│   ├── 套路库
│   ├── 爽点库
│   └── 禁忌库
├── 市场数据
│   ├── 热门作品
│   ├── 读者评价
│   └── 趋势变化
└── 创新方向
    ├── 已尝试的创新
    └── 潜在突破点
```

### 进化触发条件

| 条件 | 触发动作 | 影响范围 |
|------|----------|----------|
| 用户明确好评 | 强化该模式 | 当前Skill |
| 用户明确差评 | 弱化该模式，尝试替代 | 当前Skill |
| 质量审计低于阈值 | 全面复盘，调整策略 | 相关Skill |
| 联网发现新趋势 | 更新类型知识库 | 三级Skill |
| 累计10次同类型任务 | 生成类型专属模板 | 三级Skill |
| 系统运行满1个月 | 全面优化提示词 | 所有Skill |

## 七、优化检查清单

### 每次优化前检查
- [ ] 是否收集到足够的反馈数据？
- [ ] 问题是否已精确定位？
- [ ] 修改是否会影响其他Skill？
- [ ] 是否有备份当前版本？

### 每次优化后检查
- [ ] 输出质量是否提升？
- [ ] 是否引入新的问题？
- [ ] 协作是否仍然顺畅？
- [ ] 用户满意度是否提高？

## 八、版本管理

### 提示词版本命名
```
[Skill名称]_v[主版本].[次版本].[修订号]

示例：
- 世界观构造师_v1.0.0 → 初始版本
- 世界观构造师_v1.1.0 → 增加联网策略
- 世界观构造师_v1.1.1 → 修正输出格式
- 世界观构造师_v2.0.0 → 重大重构
```

### 版本变更记录
```markdown
## 版本历史
### v1.1.0 (2026-04-27)
- 新增：联网搜索触发时机矩阵
- 优化：记忆进化触发条件
- 修复：输出格式不一致问题

### v1.0.0 (2026-04-27)
- 初始版本
```
